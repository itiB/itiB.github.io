404 notfound pageの追加とページリンクの書き方

2020-03-18 hexo

ブログを作ったものの実はかなり基本的な機能がない状態だったのでしばらくはいろいろ追加していきます．

このブログはHexoと呼ばれる静的サイトジェネレータを使用して作られています．
構成とかどうやって作るかとかはおいおい解説することにしておいて
今日はリンクが切れていた際に表示される “404 not found” ページを作った報告

あとその際にページのリンクの書き方に悩んだのでそれもメモしています．

404 not found ページを作る

Hexoはお手軽に記事を書いたりできます．
すべてMarkdown形式で書けばよしなに作ってくれます．

Hexoでページの作成

以下のコマンドを打つことでページを作成することができます．

source ディレクトリの中にMarkdown形式のページが作成されます．
作成されたファイル内のタイトルを設定し，中身を書いたら完成です

完成したものがこちら → not-found

これを作っている中でHexo内でのページリンクってどう書くの…となったのでメモしておきます

Hexoでサイト内のリンク作成

Hexo作成したページをデプロイするとデプロイ先のURLはいろいろ代わるわけで…
作成したページのなかでデプロイ先のURLをそのまま書くのは変わる可能性がありイケてません

たぶんWebサイト作ったことのあるひとからしたら常識かもしれないですが…
トップページへのリンクは / をリンク先として書けばうまく行くみたいです．
知りませんでした

記事やページの中ではMarkdownとしてリンクを書かなければならないので以下のように書けばトップページへのリンクを作成することができました．

特定のページやPostにしたい場合は / 以下を続けて書けばよさそうです．

AzureでError Pageを設定

このサイトはAzureのストレージにファイルを置くことで作成されています．
インターネット上にファイルを置いておいてくれるサービスです．

GitHubPagesだとこのブログの存在ばれるからと思ってこっちに…完全Techの人向けにもブログ作りたいけど面倒なんだよなぁ

詳しい作り方とかはまたいつか記事にするものとしてError Pageの設定方法だけ説明します．

エラードキュメントの設定

設定 → 静的なWebサイト より エラードキュメントのパスにエラー時に表示するページを設定することで勝手にやってくれます．
